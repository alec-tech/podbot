# The Signal — Complete Setup & Operations Guide

> A fully autonomous AI-powered **three-daily** podcast covering business, tech, and policy news.
> Three 10-15 minute editions. Three optional sponsor slots per episode. Zero human intervention after setup.

---

## How It Works

| Edition | Time (EST) | News Window |
|---------|-----------|-------------|
| **Morning** | 7:00 AM | Prev 6 PM → 7 AM (~14 hrs) |
| **Midday** | 1:00 PM | 7 AM → 1 PM (~7 hrs) |
| **Evening** | 5:30 PM | 1 PM → 5:30 PM (~5 hrs) |

Episode titles are dynamically generated by Claude based on the stories covered (e.g., *"The AI Arms Race Heats Up"*, *"Markets Brace for Fed Fallout"*).

**Unified episode structure (all editions, ~12 min):**
```
0:00–0:30   Pre-Intro Sponsor (optional)
0:30–1:00   Intro
1:00–1:30   Post-Intro Sponsor (optional)
1:30–4:30   Business Block (2 stories, Alex leads)
4:30–7:30   Crossover Segment (rotating guest joins)
7:30–10:30  Tech Block (2 stories, Morgan leads)
10:30–11:00 Pre-Outro Sponsor (optional)
11:00–12:00 Wrap + Signoff
```

**Hosts:** Alex Chen (business) and Morgan Lee (tech) anchor every episode, joined by one rotating crossover guest: Drew Vasquez (geopolitics), Priya Nair (AI/deep tech), Marcus Cole (policy), Sam Torres (startups), or Jordan Blake (consumer/culture).

**How the editions stay in sync:**
- Soft dedup: earlier same-day stories provide context, not hard exclusion
- Stories CAN be revisited with UPDATE framing if major new developments occur
- 3-day cooldown applies across all editions combined

## 5. Choosing Voices

### 5.1 Voice roster

All 7 hosts need voice IDs configured per TTS provider in `config/show_config.json`:

| Host | Role | Voice Character |
|------|------|-----------------|
| **Alex Chen** | Business anchor | Authoritative, measured, confident |
| **Morgan Lee** | Tech correspondent | Warm, energetic, conversational |
| **Drew Vasquez** | Geopolitics analyst | Deliberate, gravitas, intelligent |
| **Priya Nair** | AI/emerging tech | Patient, explanatory, challenges hype |
| **Marcus Cole** | Policy correspondent | Wry, cause-and-effect reasoning |
| **Sam Torres** | Startup correspondent | Energetic, founder-fluent |
| **Jordan Blake** | Consumer/culture analyst | Bridges tech to ordinary people, irreverent |

### 5.2 Setting voice IDs

Edit the `voices` section in `config/show_config.json`. Each host has a `providers` object with entries for `cartesia`, `openai`, and `elevenlabs`:

```json
"alex": {
  "providers": {
    "cartesia": { "voice_id": "your-cartesia-id", "model_id": "sonic-3", "speed": 0.95 },
    "openai": { "voice_id": "onyx", "model": "gpt-4o-mini-tts", "instructions": "..." },
    "elevenlabs": { "voice_id": "your-elevenlabs-id", "stability": 0.55, ... }
  }
}
```

- **Cartesia**: Get voice IDs from https://cartesia.ai → Voice Library
- **OpenAI**: Uses named voices: `alloy`, `coral`, `echo`, `nova`, `onyx`, `sage`, `shimmer`
- **ElevenLabs**: Get voice IDs from https://elevenlabs.io → My Voices → Voice ID

### 5.3 Test a voice

```python
# Test Cartesia
from cartesia import Cartesia
import os; from dotenv import load_dotenv; load_dotenv()

    model_id="sonic-3",
    transcript="Welcome to The Signal morning edition. I'm Alex Chen.",
    voice={"mode": "id", "id": "your-voice-id-here"},
    output_format={"container": "mp3", "sample_rate": 44100, "bit_rate": 192000},

response.write_to_file("test_alex.mp3")
```

## 9. GitHub Actions Automation

### 9.1 How the schedule works

The workflow has three cron triggers:
```yaml
- cron: '0 12  * * 1-5'    # 12 PM UTC = 7 AM EST  → Morning edition
- cron: '0 18  * * 1-5'    # 6 PM UTC  = 1 PM EST  → Midday edition
- cron: '30 22 * * 1-5'    # 10:30 PM UTC = 5:30 PM EST → Evening edition
```

The script auto-detects which edition based on UTC hour. All editions run the same `orchestrator.py` with different `--edition` flags.

### 9.2 Add GitHub Secrets

Repo → **Settings → Secrets and variables → Actions → New repository secret**

**Required:**
| Secret | Value |
|--------|-------|
| `ANTHROPIC_API_KEY` | From console.anthropic.com |
| `BUZZSPROUT_API_KEY` | From Buzzsprout account |
| `BUZZSPROUT_PODCAST_ID` | Number from Buzzsprout URL |

**TTS (at least one):**
| Secret | Value |
|--------|-------|
| `CARTESIA_API_KEY` | From cartesia.ai |
| `OPENAI_API_KEY` | From platform.openai.com |
| `ELEVENLABS_API_KEY` | From elevenlabs.io |

**Recommended:**
| Secret | Value |
|--------|-------|
| `NEWS_API_KEY` | From newsapi.org |
| `SLACK_WEBHOOK_URL` | From Slack webhook setup |
| `PODCAST_WEBSITE_URL` | Your Netlify URL |

### 9.3 Test automation

## 10. Injecting Stories

### 10.1 CLI usage

```bash
source .venv/bin/activate

# Inject into all editions (default)
python agents/inject_stories.py \
  --url "https://techcrunch.com/2026/02/24/openai-raises-40b" \
  --priority must_include \
  --by "Alec"

# Target morning only
python agents/inject_stories.py \
  --topic "TSMC Arizona fab yields" \
  --priority consider \
  --edition morning \
  --note "Lead the tech block"

# Target evening only
python agents/inject_stories.py \
  --text "Microsoft canceled three data center leases this morning." \
  --priority must_include \
  --edition evening \
  --note "Breaking — business block lead"

# View the queue
python agents/inject_stories.py --list
python agents/inject_stories.py --list --edition morning
```

### 10.2 Priority levels

| Level | Effect |
|-------|--------|
| `must_include` | Cannot be dropped. Triggers Slack alert. |
| `consider` | High-priority — in if it beats organic. |
| `background` | Context only — enriches related stories. |

### 10.3 Edition targeting

| Flag | Effect |
|------|--------|
| `--edition all` | All three editions (default) |
| `--edition morning` | 7:00 AM edition only |
| `--edition midday` | 1:00 PM edition only |
| `--edition evening` | 5:30 PM edition only |

## 12. Sponsor Management

### 12.1 Sponsor slots

Three optional sponsor slots per episode (all editions):

| Slot | Timing | Position |
|------|--------|----------|
| `pre-intro` | 0:00–0:30 | Before the show intro |
| `post-intro` | 1:00–1:30 | After intro, before business block |
| `pre-outro` | 10:30–11:00 | Before wrap and signoff |

### 12.2 Add a sponsor

Edit `data/sponsors.json`:
```json
{
  "sponsors": [
    {
      "id": "acme-corp-123",
      "name": "Acme Corp",
      "tagline": "Build better, faster.",
      "product_description": "Acme Corp is the all-in-one platform.",
      "cta": "Try free at acme.com/signal",
      "url": "https://acme.com",
      "slot": "post-intro",
      "editions": ["morning", "midday", "evening"],
      "voice": "morgan",
      "script": "Thanks to Acme Corp for sponsoring today's Signal. Build better, faster at acme.com/signal.",
      "start_date": "2026-02-24",
      "end_date": "2026-03-31",
      "active": true
    }
  ]
}
```

Or use the Editorial Dashboard to add sponsors via the UI.

### 12.3 Without a sponsor

Empty slots are skipped — no house ads, no silence. The show simply flows from one segment to the next.

## 13. Troubleshooting

**"Edition auto-detect wrong when running locally"**
Always pass `--edition` explicitly when running locally:
```bash
python orchestrator.py --edition morning
python orchestrator.py --edition midday
python orchestrator.py --edition evening
```
The clock-based auto-detect is only for the GitHub Actions environment.

**"Script too short / too long"**
The pipeline auto-expands scripts under 10 min and trims scripts over 15 min. Check logs for the final estimated duration. If consistently off, review the story count in the brief.

**"TTS provider failing"**
The fallback chain handles this automatically (Cartesia → OpenAI → ElevenLabs). Check that at least one provider has a valid API key. Logs show which provider succeeded or failed for each line.

**"FFmpeg not found on GitHub Actions"**
The workflow installs it via `apt-get`. If it fails, it's usually a temporary mirror issue — re-run the workflow.

**"Scheduled workflow not running"**
GitHub disables scheduled workflows after 60 days of repo inactivity. Push a commit or manually trigger a run to re-enable. Cron schedules can also be delayed 15-60+ minutes.

**"Episode titled 'New Episode' on Buzzsprout"**
This was a bug (now fixed). Titles are generated dynamically by Claude. If it recurs, check that `generate_metadata()` in `publisher.py` is returning a valid `TITLE` key.
